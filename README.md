# Nedbank-Hackathon
Nedbank Transaction Volume Prediction 🚀This repository contains an effort-based solution for predicting individual customer transaction volumes over a three-month window. The model attempts to target an RMSLE between 0.34 and 0.37, utilizing high-fidelity behavioral features and a robust "Outlier-Proof" training pipeline.📊 The Challenge: The goal was to predict the next_3m_txn_count for bank customers based on:Historical Transactions: A multi-year record of transaction dates, amounts, and account IDs.Demographics: Cleaned user profiles including age, location, and professional status.Temporal Context: Predicting activity for the high-volume November–January holiday period.🛠️ Tech StackPython 3.12LightGBM: Gradient Boosting for high-performance regression.Pandas & NumPy: Data manipulation and matrix operations.PyArrow: Efficient Parquet file processing.Scikit-Learn: Cross-validation and preprocessing.🏗️ Model Architecture & Strategy1. Target Transformation (The "RMSLE Secret")To minimize Root Mean Squared Logarithmic Error, the model does not predict transaction counts directly. Instead, it trains on the log-transformed target:$$y_{target} = \ln(1 + \text{count})$$This ensures the model treats a 10-transaction error on a small account with the same weight as a 100-transaction error on a large account.2. Feature Engineering: Momentum & VelocityTransaction Velocity: Capturing activity counts in the last 30, 60, and 90 days.Momentum Engine: Calculating the ratio of the most recent month's activity against the quarterly average to detect "ramping up" behavior.Seasonality Bias: A 1.04x multiplier was applied to the final output to account for the South African holiday spending spike (Dec/Jan).3. Outlier LockdownTo prevent the "Score Explosion" (where one bad prediction ruins the total RMSLE), we implemented:Hard Clipping: All predictions are mathematically capped at 2,500, the historical maximum in the training data.Log-Space Optimization: By predicting in log-space, we avoid the risk of exponential overflows that cause non-finite submission values.🚀 How to RunEnvironment Setup:Bashpip install pandas numpy lightgbm scikit-learn pyarrow
Data Placement:Place Train.csv, Test.csv, transactions_features.parquet, and demographics_clean.parquet in the root directory.Execution:Bashpython model.py
